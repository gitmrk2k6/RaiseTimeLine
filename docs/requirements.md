# 要件定義書 — RaiseTimeLine（SNS風タイムラインアプリ）

## 1. 概要・目的

### 1.1 背景

X（旧Twitter）のようなタイムライン形式でテキスト・画像を投稿・閲覧できるSNS風Webアプリケーションを開発する。
複数ユーザーが互いの投稿にいいね・コメントを行い、フォロー機能で繋がれるプラットフォームを目指す。

### 1.2 目的

- テキスト・画像を投稿し、タイムラインとして時系列に並べて閲覧できる
- 他ユーザーの投稿にいいね・コメントを付けることで交流できる
- フォロー/フォロワー関係を構築し、ユーザー同士が繋がれる

### 1.3 本書の位置づけ

RaiseTech AIエンジニアコース 学習課題として作成する。本ドキュメントは **要件定義の目次（ハブ）** として機能し、詳細は各ドキュメントに分割して管理する。

---

## 2. 想定ユーザー

| ペルソナ | 概要 | 主な利用シーン |
| --- | --- | --- |
| 受講生 | RaiseTech コースを受講している学習者 | 学習の進捗共有、同期との交流 |
| 個人利用者 | SNSとして気軽に使いたい個人 | 日常の投稿、他ユーザーのフォロー・閲覧 |

※ 複数ユーザーが同時に利用することを前提として設計する。

## 4. 非機能要件

| カテゴリ | 要件 |
| --- | --- |
| 性能 | 通常操作（ページ表示・投稿・いいね等）が2秒以内に応答すること |
| セキュリティ | パスワードはハッシュ化して保存。JWT認証により自分のデータのみ操作可能 |
| 可用性 | 学習用途のため高可用構成は対象外（単一サーバーで動作すれば可） |
| 保守性 | Controller / Service / Repository の層構造に基づく設計とする |
| 互換性 | モダンブラウザ（Chrome, Firefox, Edge の最新版）で動作 |
| ストレージ | 画像はAWS S3に保存し、アプリサーバーには保持しない |

## 9. スコープ外（今回は実施しない）

- モバイルアプリ（ネイティブアプリ）対応
- リアルタイム通知（WebSocket等でのリアルタイムフィード更新）
- DM（ダイレクトメッセージ）機能
- ハッシュタグ・トレンド機能
- 通知機能（いいね・コメント・フォローの通知）
- 外部サービス連携（Googleログイン等）
- 本番運用（スケーリング、監視、バックアップ）
